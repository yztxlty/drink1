# Security Policy

## Supported Versions

This repository uses tag-based releases. Security fixes should be applied to the latest stable branch and released with a new semantic version tag.

## Reporting a Vulnerability

If you believe you have found a security issue in Drink1, please report it privately to the maintainer instead of opening a public issue.

Include:

- A summary of the issue
- Affected files or flows
- Reproduction steps, if available
- Any screenshots, logs, or request details that help explain the risk

## What to Avoid Publicly

Do not publish:

- Access tokens, cookies, or credentials
- Private configuration files or secrets
- Exploit details that could expose users before a fix is available

## Response Expectations

The maintainer will review the report, confirm impact, and coordinate a fix on the stable branch. Once the issue is resolved, the fix will be shipped through the normal release flow.

## Scope

This policy applies to repository code, release artifacts, GitHub workflows, and documentation that could affect user safety or data privacy.
